# Contributing to QuestCoin

Thank you for your interest in contributing to QuestCoin!

## How to Contribute

### Reporting Bugs
- Use GitHub Issues
- Include detailed steps to reproduce
- Provide contract addresses and transaction hashes if applicable

### Feature Requests
- Open an issue describing the feature
- Explain the use case and benefits

### Code Contributions

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly on testnet
5. Submit a pull request

### Smart Contract Changes
- All contract changes must be reviewed
- Include test coverage
- Document any new functions
- Consider gas optimization

### Frontend Changes
- Maintain dark theme consistency
- Test wallet connection flows
- Ensure mobile responsiveness

## Code Style

### Solidity
- Use NatSpec comments
- Follow OpenZeppelin patterns
- Keep functions focused and small

### JavaScript
- Use async/await for promises
- Handle errors gracefully
- Add meaningful variable names

## Security

If you discover a security vulnerability, please report it privately rather than opening a public issue.

## Questions?

Open a discussion on GitHub or reach out through Farcaster.
